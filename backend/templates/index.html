Why Email Header Forensics?

Email remains the most exploited attack vector in cybersecurity. Phishing emails often bypass visual detection, making header analysis critical for identifying spoofed domains, forged authentication, and suspicious routing paths.

This platform extracts and analyzes hidden metadata from email headers to provide actionable forensic insights.

How to Extract Email Headers

Email headers contain hidden routing, authentication, and delivery metadata. Follow the steps below based on your email platform to extract the complete header for forensic analysis.

🪟

Windows Mail Clients

  • Open the email message
  • Select File → Properties
  • Copy content from Internet Headers
🍎

Apple Mail (macOS)

  • Open the email
  • Go to View → Message → All Headers
  • Copy the complete header section
📧

Gmail (Web)

  • Open the email
  • Click ⋮ → Show original
  • Copy the full header text
📨

Outlook (Web)

  • Open the email
  • Select ⋯ → View → Message details
  • Copy the message headers

What the Platform Analyzes

Authentication Integrity

Verifies SPF, DKIM, and DMARC alignment to detect spoofed or forged sender identities.

Routing Path Analysis

Traces each mail server hop to identify abnormal paths, relay abuse, or multi-country threat patterns.

IP Geolocation Intelligence

Maps originating and relay IP addresses to geographic regions for threat attribution.

Risk Scoring & Verdict

Combines authentication, routing, and heuristics into a quantified risk score with a clear verdict.
